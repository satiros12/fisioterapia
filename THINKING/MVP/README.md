# MVP Plan

## Minimal Viable Product Features

### Core Features
- Patient registration with basic profile information
- Exercise library with static exercise descriptions
- Progress tracking with manual EVA input
- Basic reporting functionality

### Data Structure
- Patients: Name, age, condition, contact info
- Exercises: Name, description, images, difficulty level
- Progress: Date, exercises completed, EVA score

### Technology Stack
- Backend: Flask (lightweight, easy to test)
- Database: SQLite (simple, good for MVP)
- Frontend: Basic HTML/CSS/JavaScript

### Mock Data Structure


### MVP Features
- Patient CRUD operations
- Exercise catalog with filtering
- Basic progress tracking
- Simple reporting
- Authentication (basic)

### Success Criteria
- 5 patients can be registered
- 10 exercises can be viewed
- Progress tracking works for 1 patient
- Basic reports can be generated
